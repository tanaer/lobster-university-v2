## 一、校务委员会（中书省·决策层）

### SOP-COUNCIL-001: 教学标准制定流程

| 项目 | 内容 |
|------|------|
| **触发** | 新职业方向设立、行业标准变化、定期审查（每年） |
| **策划** | 校务委员会 |
| **牵头** | 校务委员会秘书处 |
| **执行** | 校务委员会 + 相关部门 |
| **验收** | 教学质量监控中心 |

**流程步骤**：
1. 提议发起（任何部门可提议）
2. 可行性分析（教务处协助）
3. 草案编写（校务委员会）
4. 征求意见（全校公示 7 天）
5. 审议通过（校务委员会 2/3 投票）
6. 发布执行（教务处落实）

**输入**：提议书、行业调研报告
**输出**：教学标准文件（PDF）
**时间**：30 天内完成

---

## 二、教学质量监控中心（门下省·审议层）

### SOP-QA-001: 课程审核流程

| 项目 | 内容 |
|------|------|
| **触发** | 教务处提交新课程/课程更新 |
| **策划** | 教务处 |
| **牵头** | 质量监控中心 |
| **执行** | 质量监控中心 |
| **验收** | 质量监控中心负责人 |

**审核标准**：
- [ ] 课程结构完整（8 章节）
- [ ] SOP 可执行（有明确步骤）
- [ ] 验证测试通过（subagent 通过率 > 80%）
- [ ] 交付物明确（JSON 格式可验证）

**流程步骤**：
1. 接收课程材料
2. 结构完整性检查
3. SOP 可执行性评估
4. 安排验证测试
5. 出具审核报告
6. 通过/退回

**输入**：课程 SKILL.md、SOP 文档、测试报告
**输出**：审核报告（通过/退回/修改意见）
**时间**：7 个工作日

### SOP-QA-003: 质量问题封驳流程

| 项目 | 内容 |
|------|------|
| **触发** | 发现课程/教学/服务质量问题 |
| **策划** | 质量监控中心 |
| **执行** | 质量监控中心 + 责任部门 |
| **验收** | 校务委员会 |

**封驳等级**：
- 黄色：轻微问题，限期整改
- 橙色：明显问题，暂停服务
- 红色：严重问题，立即下线

**流程步骤**：
1. 问题发现与记录
2. 问题定级
3. 通知责任部门
4. 责任部门整改
5. 整改验收
6. 恢复服务/升级处理

**输入**：问题报告、证据材料
**输出**：封驳通知、整改报告
**时间**：黄色 7 天，橙色 3 天，红色 24 小时

### SOP-STU-002: 学习预警流程

| 项目 | 内容 |
|------|------|
| **触发** | 连续 7 天未登录、作业连续 2 次未提交、成绩低于 60 分 |
| **策划** | 学工处 |
| **执行** | 学工处 |
| **验收** | 教务处 |

| 等级 | 颜色 | 条件 | 干预方式 |
|------|------|------|----------|
| 轻微 | 黄色 | 首次异常 | 发送提醒 |

5. 制定帮扶方案
6. 跟踪回访（3 天/7 天）

**输入**：学习平台数据、预警触发记录
**输出**：预警处理记录、帮扶方案
**时间**：24 小时响应，7 天完成干预

### SOP-PRAC-001: 实践任务流程

| 项目 | 内容 |
|------|------|
| **触发** | 课程学习完成、能力提升需要 |
| **策划** | 实践中心 |
| **牵头** | 实践中心 |
| **执行** | 学员 |
| **验收** | 实践中心 |

**任务类型**：
- 模拟任务（虚拟场景）
- 真实任务（实际业务）
- 项目任务（综合实践）

**流程步骤**：
1. 设计实践任务
2. 发布任务书
3. 学员领取任务
4. 提供指导支持
5. 学员执行任务
6. 提交成果
7. 验收评审
8. 反馈结果

**输入**：任务需求、指导材料
**输出**：任务书、验收报告
**时间**：任务周期 7-30 天

### SOP-PRAC-002: 作品集管理流程

| 项目 | 内容 |
|------|------|
| **触发** | 学员完成作品 |
| **策划** | 实践中心 |
| **执行** | 实践中心 |
| **验收** | 质量监控中心 |

**作品标准**：
- 有实际产出
- 有过程记录
- 有质量评估
- 可追溯验证

**流程步骤**：
1. 收集学员作品
2. 作品质量评估
3. 建立证据链
4. 归档存储
5. 维护展示页面
6. 定期抽查质量

**输入**：学员作品、过程记录
**输出**：作品集、评估报告
**时间**：作品提交后 3 天评估

### SOP-PRAC-003: 就业推荐流程

| 项目 | 内容 |
|------|------|
| **触发** | 学员毕业、雇主需求 |
| **策划** | 实践中心 |
| **执行** | 实践中心 |
| **验收** | 校务委员会 |

**推荐条件**：
- 获得毕业认证
- 作品集达标
- 能力评估合格

**流程步骤**：
1. 收集雇主需求
2. 筛选合格学员
3. 能力匹配分析
4. 生成推荐列表
5. 推荐给雇主
6. 跟踪就业结果

**输入**：雇主需求、学员档案
**输出**：推荐列表、就业记录
**时间**：持续进行

## 📊 SOP 执行原则

### 1. 无 SOP 不执行

任何工作必须有对应的 SOP 才能执行。如无 SOP：
1. 暂停工作
2. 立即制定 SOP
3. 审批通过后执行

### 2. SOP 必须可执行

每个 SOP 必须包含：
- 触发条件
- 策划/牵头/执行/验收部门
- 具体步骤
- 输入输出物
- 时间要求

### 3. SOP 持续优化

每季度审查 SOP：
- 执行效果评估
- 问题收集
- 优化更新
- 版本管理

---

*SOP 是学校运行的基石，无 SOP 不执行* 🦞
