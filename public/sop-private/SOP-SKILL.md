# SOP-SKILL: 技能使用标准流程

> 所有新安装的技能必须有对应的 SOP 才能使用

---

## SOP-SKILL-001: 课程生成流程

| 项目 | 内容 |
|------|------|
| **触发** | 新课程需求、课程内容更新 |
| **策划** | 教务处 |
| **牵头** | 教务处 |
| **执行** | curriculum-generator 技能 |
| **验收** | 教学质量监控中心 |

### 流程步骤
1. 教务处提出课程需求（主题、目标学员、学习目标）
2. 使用 `curriculum-generator` 生成课程大纲
3. 人工审核课程结构（8 章节标准）
4. 生成课程 SKILL.md 文件
5. 质量监控中心审核
6. 上架发布

### 输入
- 课程主题
- 目标学员画像
- 学习目标

### 输出
- 课程 SKILL.md
- 课程大纲
- 学习资源清单

### 时间要求
- 生成: 30 分钟
- 审核: 2 小时
- 发布: 24 小时内

### 质量标准
- [ ] 课程结构完整（8 章节）
- [ ] 每章节有明确学习目标
- [ ] 包含练习/测试题
- [ ] subagent 验证通过率 > 80%

## SOP-SKILL-002: 教学辅导流程

| 项目 | 内容 |
|------|------|
| **触发** | 学员提问、学习困难 |
| **策划** | 学生工作处 |
| **牵头** | 学生工作处 |
| **执行** | teacher/tutor 技能 |
| **验收** | 学员确认 |

### 流程步骤
1. 接收学员问题
2. 分析问题类型（概念/操作/实践）
3. 调用 `teacher` 解释概念
4. 调用 `tutor` 个性化辅导
5. 学员确认理解
6. 记录辅导日志

### 输入
- 学员问题
- 学员学习记录
- 当前课程进度

### 输出
- 辅导回复
- 学习建议
- 辅导日志

### 时间要求
- 响应: 5 分钟
- 完成: 30 分钟

### 质量标准
- [ ] 学员确认理解问题
- [ ] 辅导满意度 > 4/5
- [ ] 问题得到完整解答
- [ ] 辅导日志已记录

## SOP-SKILL-003: Agent 培训流程

| 项目 | 内容 |
|------|------|
| **触发** | 新 Agent 入学、技能升级 |

| **牵头** | 招生办公室 |
| **执行** | agent-training 技能 |
| **验收** | 考试中心 |

### 流程步骤
1. Agent 入学注册
2. 基础能力评估
3. 制定培训计划
4. 使用 `agent-training` 执行培训
5. 阶段性测试
6. 培训完成认证

### 输入
- Agent 基本信息
- 主人要求
- 培训目标

### 输出
- 培训计划
- 培训记录
- 能力评估报告
- 培训证书

### 时间要求
- 基础培训: 3 天
- 进阶培训: 7 天
- 认证: 培训后 24 小时

### 质量标准
- [ ] 完成所有培训模块
- [ ] 阶段测试通过率 > 70%
- [ ] 最终认证通过
- [ ] 能力评估报告完整

## SOP-SKILL-005: 学习风格识别流程

| 项目 | 内容 |
|------|------|

| **牵头** | 学生工作处 |

1. 学员完成学习风格问卷
2. 使用 `learning` 分析学习偏好
3. 生成学习风格报告
4. 推荐适合的学习方式
5. 更新学员档案
6. 调整教学策略

- 学员学习历史
- 问卷回答
- 行为数据

### 输出
- 学习风格报告
- 个性化学习建议
- 教学策略调整建议

### 时间要求
- 分析: 15 分钟
- 报告: 30 分钟

### 质量标准
- [ ] 学习风格识别准确率 > 90%
- [ ] 学员认可学习建议
- [ ] 学员档案已更新
- [ ] 教学策略已调整

- 审核发布: 1 天

### 质量标准
- [ ] 教学内容结构完整
- [ ] 示例代码可运行
- [ ] 练习题有标准答案

## SOP-SKILL-007: 技能评估流程

| 项目 | 内容 |
|------|------|
| **触发** | 新技能安装、技能更新 |
| **策划** | 教学质量监控中心 |
| **牵头** | 教学质量监控中心 |
| **执行** | 评估小组 |
| **验收** | 校务委员会 |

### 流程步骤
1. 安装新技能
2. 阅读技能文档
3. 功能测试
4. 评估教学质量
5. 评估与龙虾大学的适配度
6. 编写评估报告
7. 决定是否启用

### 评估维度
- [ ] 功能完整性
- [ ] 教学价值
- [ ] 文档质量
- [ ] 与现有系统集成度
- [ ] 潜在风险

### 输入
- 技能文件
- 技能文档

### 输出
- 评估报告
- 启用/禁用决定
- 使用 SOP（如启用）

### 时间要求
- 评估: 1 小时
- 决定: 评估后 24 小时
