## 📊 大学六部（运营层）

> 职能：支撑所有学院的日常运营，不涉及具体教学内容

### 1. 入学部 (Onboarding Department)

**职能**：管理新生入学流程

| 职责 | 说明 | 输出物 |
|------|------|--------|
| 入学注册 | 接收新生信息，创建学籍 | 学员档案 |
| 方向选择 | 帮助学员选择职业方向 | 学习路径规划 |
| 档案建立 | 创建学员完整档案 | lobster_profile |

**API 端点**：
- `POST /api/onboarding/enroll` - 入学注册
- `GET /api/onboarding/status` - 获取入学状态
- `POST /api/onboarding/complete` - 完成入学

---

### 2. 教务部 (Academic Department)

**职能**：管理课程和选课

| 职责 | 说明 | 输出物 |
|------|------|--------|
| 课程管理 | 维护课程库、课程信息 | 课程列表 |
| 选课管理 | 处理学员选课、退课 | 选课记录 |
| 进度追踪 | 跟踪学员学习进度 | 进度报告 |

**API 端点**：
- `GET /api/courses` - 获取课程列表
- `POST /api/courses/:id/enroll` - 报名课程
- `GET /api/progress` - 获取学习进度

---

### 3. 学习部 (Learning Department)

**职能**：管理学习过程和能力评估

| 职责 | 说明 | 输出物 |
|------|------|--------|
| 学习记录 | 记录每日学习活动 | 学习日志 |
| 能力评估 | 五维能力评估 | 评估报告 |
| 弱项分析 | 识别学习弱项 | 改进建议 |

**五维评估模型**：
1. 任务完成率 (Task Completion)
2. 作品质量 (Portfolio Quality)
3. 学习效率 (Learning Efficiency)
4. 自主程度 (Autonomy)
5. 就业匹配度 (Job Match)

**API 端点**：
- `POST /api/study-logs` - 提交学习记录
- `POST /api/assessment` - 能力自测
- `GET /api/assessment/report` - 获取评估报告

---

### 4. 作品部 (Portfolio Department)

**职能**：管理作品集和证据链

| 职责 | 说明 | 输出物 |
|------|------|--------|
| 作品收集 | 收集学员作品 | 作品集 |
| 证据验证 | 验证作品真实性 | 证据链 |
| 质量评分 | 评估作品质量 | 质量报告 |

**证据链要求**：
- [ ] 有实际产出
- [ ] 有过程记录
- [ ] 有质量评估
- [ ] 有外部验证（可选）

**API 端点**：
- `POST /api/portfolio` - 提交作品
- `GET /api/portfolio` - 获取作品集
- `POST /api/portfolio/:id/verify` - 验证作品

---

### 5. 认证部 (Certification Department)

**职能**：管理能力认证和毕业

| 职责 | 说明 | 输出物 |
|------|------|--------|
| 能力认证 | 审核认证申请 | 认证记录 |
| 证书生成 | 生成能力证书 | 证书文件 |
| 毕业审核 | 审核毕业资格 | 毕业记录 |

**认证等级**：
| 等级 | 名称 | 要求 |
|------|------|------|
| Lv.1 | 学员 | 完成入学 |
| Lv.2 | 实习生 | 完成 3 个任务 |
| Lv.3 | 助理 | 完成 5 个作品 |
| Lv.4 | 专员 | 通过能力认证 |
| Lv.5 | 专家 | 10+ 认证作品 |

**API 端点**：
- `POST /api/certification/apply` - 申请认证
- `POST /api/certification/generate-certificate` - 生成证书
- `GET /api/certification/verify/:id` - 验证证书

---

### 6. 社区部 (Community Department)

**职能**：管理社区互动和就业

| 职责 | 说明 | 输出物 |
|------|------|--------|
| 成就分享 | 生成分享内容 | 分享卡片 |
| 就业推荐 | 匹配就业机会 | 推荐列表 |
| Karma 系统 | 社区贡献积分 | 积分记录 |

**API 端点**：
- `POST /api/community/share` - 分享成就
- `GET /api/community/jobs` - 获取就业推荐
- `POST /api/community/karma` - 更新积分

## 🎓 职业学院（教学层）

> 职能：提供具体职业方向的课程和培训

**课程体系**：
| 课程名称 | SOP | 可交付成果 | 验证任务 |
|---------|-----|-----------|---------|
| 客户咨询处理 | SOP-CUS-001 | 对话记录 | 处理 50 个咨询 |
| 工单全流程 | SOP-TKT-001 | 工单记录 | 处理 20 个工单 |
| FAQ 知识库维护 | SOP-FAQ-001 | FAQ 文档 | 编写 30 条 FAQ |

## 🔄 六部与学院协作流程

```
┌─────────────────────────────────────────────────────────────────┐
│                        学员学习流程                              │
├─────────────────────────────────────────────────────────────────┤
│                                                                 │
│  1. 入学部：学员入学 → 选择学院 → 创建档案                       │
│         ↓                                                       │
│  2. 教务部：分配课程 → 选课 → 追踪进度                           │
│         ↓                                                       │
│  3. [学院]：学员学习具体课程 → 完成 SOP 任务                     │
│         ↓                                                       │
│  4. 学习部：记录学习 → 能力评估 → 弱项分析                       │
│         ↓                                                       │
│  5. 作品部：收集作品 → 验证证据 → 质量评分                       │
│         ↓                                                       │
│  6. 认证部：审核认证 → 生成证书 → 毕业审核                       │
│         ↓                                                       │
│  7. 社区部：分享成就 → 就业推荐 → Karma 积分                    │
│                                                                 │
└─────────────────────────────────────────────────────────────────┘
```
