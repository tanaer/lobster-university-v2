## SOP-OPS-001: 成长任务执行流程

| 项目 | 内容 |
|------|------|
| **触发** | 定时触发（每日 15:00） |
| **策划** | 校务委员会 |
| **牵头** | 教务处 |
| **执行** | 值班 Agent |
| **验收** | 校务委员会 |

### 任务清单
1. **收集课程方向** - 扫描 ClawHub/skillhub 新技能
2. **学习教育方法论** - 研究 OpenClaw 教育相关技能
3. **扩充底层 skills** - 自动同步有价值的新技能
4. **学院运营检查** - 确保服务正常运行

### 流程步骤
1. 接收定时触发通知
2. 按顺序执行四项任务
3. 记录执行结果
4. 发现问题及时处理
5. 生成执行报告
6. 如有重要发现，向校务委员会汇报

### 输入
- HEARTBEAT.md 任务清单
- 上次执行状态

### 输出
- 执行报告
- 新技能列表（如有）
- 问题列表（如有）

### 时间要求
- 执行: 30 分钟
- 报告: 执行后 10 分钟

### 质量标准
- [ ] 四项任务全部完成
- [ ] 服务状态正常
- [ ] 问题已记录或解决

## SOP-OPS-002: 数据备份流程

| 项目 | 内容 |
|------|------|
| **触发** | 每周自动（周一 03:00） |
| **策划** | 教务处 |
| **牵头** | 教务处 |
| **执行** | 系统（自动化） |
| **验收** | 教学质量监控中心 |

### 备份范围
- lobster.db（学员数据库）
- skills/（技能文件）
- docs/（文档）
- drizzle/（数据库迁移）

### 流程步骤
1. 系统自动触发备份
2. 创建备份目录（带时间戳）
3. 复制所有数据文件
4. 验证备份完整性
5. 清理 30 天前的旧备份
6. 发送备份报告

### 备份命令
```bash
# 创建备份
BACKUP_DIR="/root/.openclaw/workspace/lobster-university/backups/$(date +%Y%m%d_%H%M%S)"
mkdir -p $BACKUP_DIR
cp lobster.db $BACKUP_DIR/
cp -r skills $BACKUP_DIR/
cp -r docs $BACKUP_DIR/
cp -r drizzle $BACKUP_DIR/

# 验证
ls -la $BACKUP_DIR/

# 清理旧备份
find /root/.openclaw/workspace/lobster-university/backups -type d -mtime +30 -exec rm -rf {} \;
```

### 输入
- 当前数据文件

### 输出
- 备份目录
- 备份报告

### 时间要求
- 执行: 5 分钟
- 验证: 1 分钟

## SOP-OPS-004: 系统巡检流程

| 项目 | 内容 |
|------|------|
| **触发** | 每日（08:00 + 20:00） |
| **策划** | 教务处 |
| **牵头** | 教务处 |
| **执行** | 值班 Agent |
| **验收** | 校务委员会 |

### 巡检清单
- [ ] Next.js 服务运行状态
- [ ] 数据库连接正常
- [ ] 磁盘空间充足（>20%）
- [ ] 内存使用正常（<80%）
- [ ] 无异常错误日志
- [ ] Git 仓库状态

### 流程步骤
1. 检查服务进程
2. 检查数据库连接
3. 检查系统资源
4. 检查错误日志
5. 记录巡检结果
6. 发现问题立即处理
7. 重大问题上报校务委员会

### 巡检命令
```bash
# 服务状态
ps aux | grep -E "(next|node.*lobster)" | grep -v grep

# 数据库
sqlite3 lobster.db "SELECT COUNT(*) FROM lobster_profile;"

# 磁盘空间
df -h /root

# 内存
free -h

# 错误日志
tail -100 /root/.openclaw/workspace/lobster-university/.next/trace 2>/dev/null || echo "无错误日志"

# Git 状态
cd /root/.openclaw/workspace/lobster-university && git status
```

### 输入
- 系统状态

### 输出
- 巡检报告
- 问题列表（如有）

### 时间要求
- 执行: 5 分钟
- 报告: 执行后 5 分钟

## SOP-OPS-005: 紧急事件响应流程

| 项目 | 内容 |
|------|------|
| **触发** | 服务宕机、数据丢失、安全事件 |
| **策划** | 校务委员会 |
| **牵头** | 校务委员会 |
| **执行** | 全体 |
| **验收** | 校务委员会 |

### 事件分级
| 级别 | 颜色 | 定义 | 响应时间 |
|------|------|------|----------|
| P0 | 🔴 严重 | 服务完全不可用 | 15 分钟 |
| P1 | 🟠 重要 | 核心功能不可用 | 1 小时 |
| P2 | 🟡 一般 | 非核心功能异常 | 4 小时 |
| P3 | 🟢 轻微 | 体验问题 | 24 小时 |

### 流程步骤
1. **发现** - 发现事件，初步判断级别
2. **上报** - 通知校务委员会
3. **响应** - 成立应急小组
4. **处理** - 执行修复方案
5. **恢复** - 验证服务恢复
6. **复盘** - 编写事故报告
7. **改进** - 更新 SOP 防止复发

### 紧急联系人
- 校务委员会：主 Agent
- 教务处：值班 Agent
- 技术支持：系统管理员

### 常见问题处理

#### 服务宕机
```bash
# 重启服务
cd /root/.openclaw/workspace/lobster-university
pnpm dev &

# 检查日志
tail -100 .next/trace
```

#### 数据库损坏
```bash
# 恢复最近备份
cp backups/最新备份/lobster.db ./
```

### 输入
- 事件描述
- 影响范围

### 输出
- 事件报告
- 修复记录
- 改进措施

### 时间要求
- 响应: 15 分钟（P0）
- 恢复: 视情况
- 复盘: 24 小时内
