# SOP-PRACTICE-001: 模拟实训流程

> 实践教学中心核心职责：通过模拟实训提升学员实战能力

---

## 一、部门职责

| 角色 | 部门 | 职责 |
|------|------|------|
| **策划** | 实践教学中心 | 设计实训场景、制定训练方案 |
| **牵头** | 实践教学中心 | 组织实训、协调资源 |
| **执行** | 实践教学中心 | 部署实训环境、指导学员 |
| **验收** | 教学质量监控中心 | 评估实训效果、确认能力提升 |

---

## 二、实训类型

### 2.1 基础技能实训
- **目标**：掌握单一技能的标准执行流程
- **场景**：理想条件下完成任务
- **示例**：按 SOP 完成一次 GA4 配置

### 2.2 复杂场景实训
- **目标**：在多变条件下解决问题
- **场景**：条件变化、资源受限、时间压力
- **示例**：API 限流时如何完成数据抓取

### 2.3 故障排查实训
- **目标**：遇到问题时诊断和解决
- **场景**：报错、超时、结果异常
- **示例**：Server Action 失败时如何排查

### 2.4 综合项目实训
- **目标**：多技能组合完成真实项目
- **场景**：端到端交付
- **示例**：从需求到上线的完整网站开发

## 三、实训流程

```
┌─────────────────────────────────────────────────────────────┐
│                     模拟实训流程                             │
└─────────────────────────────────────────────────────────────┘

1. 场景设计
   ├── 确定实训目标
   ├── 设计初始条件
   ├── 设计变化条件（随机触发）
   └── 设计验收标准

2. 环境部署
   ├── 准备实训资源
   ├── 配置监控和日志
   └── 部署自动化评分系统

3. 学员执行
   ├── 接收任务
   ├── 执行 SOP
   ├── 遇到问题 → 进入故障处理流程
   └── 提交成果

4. 自动评分
   ├── 检查输出是否符合预期
   ├── 检查过程是否合规
   ├── 生成评分报告
   └── 反馈给学员

5. 复盘总结
   ├── 分析成功/失败原因
   ├── 更新知识库
   └── 优化 SOP
```

## 四、变化条件设计

实际执行时客观条件会变化，实训必须模拟这些：

| 变化类型 | 示例 | 应对能力 |
|----------|------|----------|
| **资源限制** | API 限流、磁盘满、内存不足 | 资源管理、降级方案 |
| **网络问题** | 超时、断网、DNS 失败 | 重试、缓存、离线方案 |
| **依赖故障** | 第三方服务宕机 | 熔断、备用方案 |
| **数据异常** | 格式错误、缺失字段 | 数据清洗、容错处理 |
| **权限问题** | Token 过期、权限不足 | 认证刷新、权限申请 |
| **时间压力** | 任务超时、 deadline 临近 | 优先级、快速决策 |

## 五、故障处理 SOP

### 5.1 故障分级

| 级别 | 描述 | 处理方式 |
|------|------|----------|
| L1 | 轻微问题，可自行解决 | 学员自主处理，记录日志 |
| L2 | 中等问题，需要指导 | 触发提示，提供解决思路 |
| L3 | 严重问题，无法继续 | 暂停实训，人工介入 |

### 5.2 故障处理流程

```
遇到问题
    │
    ▼
┌─────────────┐
│ 尝试自主解决 │ ← 使用已有知识和技能
└──────┬──────┘
       │
       ├─ 解决 → 记录方案 → 继续实训
       │
       └─ 未解决 → 查看知识库
                      │
                      ├─ 找到方案 → 尝试 → 记录
                      │
                      └─ 未找到 → 触发提示
                                        │
                                        ├─ 提示有效 → 继续
                                        │
                                        └─ 仍无效 → 人工介入
```

### 5.3 反馈上报机制

当学员遇到无法解决的问题时：

1. **自动收集信息**
   - 当前执行步骤
   - 错误日志
   - 环境状态
   - 已尝试的解决方案

2. **生成反馈报告**
   ```json
   {
     "task_id": "xxx",
     "step": 3,
     "error": "API rate limit exceeded",
     "attempts": ["retry", "reduce batch size"],
     "logs": "...",
     "timestamp": "2026-03-13T14:50:00Z"
   }
   ```

3. **提交到教学质量监控中心**
   - 自动创建 Issue
   - 标记优先级
   - 通知相关教授

## 六、验收标准

### 6.1 技能掌握度评估

| 等级 | 标准 | 说明 |
|------|------|------|
| S | 完美执行，能处理所有变化 | 优秀 |
| A | 标准执行，能处理常见变化 | 良好 |
| B | 标准执行，遇到变化需提示 | 合格 |
| C | 执行有偏差，需多次指导 | 待改进 |
| D | 无法完成基本执行 | 不合格 |
