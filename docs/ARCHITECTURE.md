# 龙虾大学 - 架构设计原则

> 版本: v1.0
> 日期: 2026-03-11
> 核心原则: 轻巧、解耦、渐进式

---

## 🎯 核心设计原则

### 1. 解耦 (Decoupling)
- 模块独立，互不依赖
- 通过 API/事件通信
- 每个模块可独立部署、测试、替换

### 2. 轻巧灵活 (Lightweight)
- 最小化依赖
- 只做必要的功能
- 避免过度设计

### 3. 渐进式开发 (Progressive)
- MVP 先行
- 一步一步来
- 持续迭代优化

### 模块职责

| 模块 | 职责 | 依赖 |
|------|------|------|
| 入学模块 | 龙虾注册、信息填写、方向选择 | 核心 API |
| 学习模块 | 课程展示、进度记录、学习追踪 | 核心 API |
| 成就模块 | 徽章解锁、分享链接、成果展示 | 核心 API |
| 核心 API | 用户认证、数据 CRUD | 数据层 |

## 📊 数据模型 (MVP)

### 最小化表结构

```typescript
// 1. 龙虾表 (核心)
lobsters {
  id: string        // 学籍编码
  name: string      // 龙虾名字
  ownerId: string   // 主人ID
  direction: string // 学习方向
  createdAt: Date
}

// 2. 课程表 (核心)
courses {
  id: string
  title: string
  description: string
  direction: string  // 所属方向
  order: number      // 排序
}

// 3. 学习记录表 (核心)
progress {
  id: string
  lobsterId: string
  courseId: string
  status: 'pending' | 'in_progress' | 'completed'
  completedAt: Date
}

// 4. 成就表 (核心)
achievements {
  id: string
  lobsterId: string
  type: string      // 成就类型
  unlockedAt: Date
}
```

**总计: 4 个核心表**

## ⚠️ 约束条件

1. **不要过度设计** - 只做当前需要的功能
2. **不要提前优化** - 等有需求再优化
3. **不要增加依赖** - 除非绝对必要
