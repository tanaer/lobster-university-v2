# 龙虾大学 - 课程设计原则

> **核心原则**: 职业教育的目的是**就业**，不是学习工具

---

## ❌ 错误的课程设计（学工具）

```
电商运营专业:
├── Excel 高级函数应用      ← 学工具
├── Python 数据处理基础     ← 学工具
├── SQL 数据库操作         ← 学工具
└── 数据可视化最佳实践      ← 学工具

问题：学完不知道能干什么工作
```

## ✅ 正确的课程设计（能上岗）

```
电商运营专业:
├── 能独立完成店铺日常运营    ← 工作任务
├── 能生成销售日报          ← 可交付成果
├── 能处理客户咨询          ← 工作任务
└── 能完成竞品分析报告       ← 可交付成果

结果：学完就能上岗
```

---

## 🎯 课程设计方法论

### 1. 从岗位 JD 逆向设计

```
招聘 JD 写什么 → 课程就教什么

JD: "负责店铺日常运营，跟踪销售数据"
课程: 能独立完成店铺日常运营，能生成销售日报

JD: "处理客户售前售后问题"
课程: 能处理客户咨询，能生成客服周报
```

### 2. 每门课 = 一个工作任务

| ❌ 错误（学知识） | ✅ 正确（做任务） |
|-----------------|-----------------|
| 学习 Excel 函数 | 能生成销售报表 |
| 学习 Python 语法 | 能自动化处理数据 |
| 学习 SQL 查询 | 能提取业务数据 |
| 学习数据可视化 | 能制作分析图表 |

### 3. 评估标准 = 可交付成果

```
不是考"知不知道"，而是考"能不能做出来"

❌ 考试: 什么是 SQL JOIN？
✅ 任务: 从数据库提取上月销售数据并生成报告

❌ 考试: Python 有哪些数据类型？
✅ 任务: 写脚本自动处理 1000 条商品数据
```

## 📋 重新设计的专业课程

### 专业 1: 电商运营专员

**就业目标**: 能独立负责一个淘宝/拼多多店铺的日常运营

**核心能力（学完能做什么）**:
| 序号 | 能力 | 可交付成果 |
|------|------|-----------|
| 1 | 能完成商品上架 | 10 个商品完整上架 |
| 2 | 能生成销售日报 | 每日销售数据报告 |
| 3 | 能处理客户咨询 | 客服对话记录 + 满意度 |
| 4 | 能完成竞品分析 | 竞品分析报告 |
| 5 | 能策划促销活动 | 活动方案 + 执行记录 |
| 6 | 能优化商品标题 | 标题优化前后对比 |

**毕业作品集**:
- 一份完整的店铺运营月报
- 一份竞品分析报告
- 一份促销活动复盘

## 🔄 课程结构对比

### 旧结构（学工具）
```
课程 → 章节 → 知识点 → 考试
```

### 新结构（能上岗）
```
专业 → 工作任务 → 可交付成果 → 作品集
```

---

## 📊 评估体系

### 不考"知道什么"，考"能做什么"

| 评估维度 | 旧方式 | 新方式 |
|---------|--------|--------|
| 知识 | 选择题考试 | ❌ 取消 |
| 技能 | 上机操作 | ✅ 实际任务 |
| 能力 | 综合测试 | ✅ 作品集 |
| 就业 | 推荐信 | ✅ 作品集 + 能力证明 |

### 毕业标准

```
不是"学完所有课程"，而是"完成所有可交付成果"

毕业条件:
✅ 完成 6 个核心能力的可交付成果
✅ 提交完整的毕业作品集
✅ 通过实际任务测试
```

---

## 💡 实施建议

1. **每门课开头先说"学完能做什么"**
2. **每门课结尾要求"提交可交付成果"**
3. **毕业必须有"作品集"而非"成绩单"**
4. **课程更新跟随招聘 JD 变化**

---

**核心原则**:
> 职业教育的目的不是学习，而是就业
> 课程设计的标准不是"教了什么"，而是"能做什么"
