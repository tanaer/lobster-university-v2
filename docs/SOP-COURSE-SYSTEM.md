# SOP-COURSE-SYSTEM: 课程体系设计 SOP

> 教务处主持，IT部门配合
> 创建时间：2026-03-14
> 目的：将零散课程组合成能解决具体问题或胜任具体岗位的课程体系

---

## 一、课程体系定义

课程体系 ≠ 课程分类。课程体系是面向一个具体岗位或具体问题的学习路径。

| 维度 | 课程分类（旧） | 课程体系（新） |
|------|--------------|--------------|
| 组织方式 | 按技术领域分 | 按岗位/问题分 |
| 目标 | 学会某个工具 | 胜任某个岗位 |
| 示例 | "数字营销"分类下26门课 | "小红书运营岗"体系含5门课 |
| 学员视角 | 不知道选哪门 | 跟着路径学就能上岗 |

---

## 二、课程体系设计流程

### Step 1: 确定岗位/问题

从以下来源确定课程体系方向：
- 招聘网站热门岗位 JD
- 学员反馈的实际需求
- 行业趋势分析

### Step 2: 逆向拆解能力要求

```
岗位 JD → 核心能力 → 对应课程 → 学习顺序
```

### Step 3: 组合课程路径

每个课程体系包含：
- 体系名称（面向岗位/问题）
- 体系描述（学完能做什么）
- 课程列表（按学习顺序排列）
- 预计学习时长
- 前置要求

### Step 4: 录入数据库

## 三、首批课程体系设计

### 体系 1: 新媒体运营岗
- 目标：能独立运营品牌的新媒体矩阵
- 课程：小红书运营 → 抖音策略 → 微信公众号 → B站内容 → 社媒策略
- 时长：8周

### 体系 2: 电商运营岗
- 目标：能独立运营电商店铺并完成大促策划
- 课程：国内电商全平台 → 直播电商 → 私域运营 → 增长黑客
- 时长：6周

### 体系 3: 跨境电商岗
- 目标：能独立完成品牌出海和海外电商运营
- 课程：跨境电商出海 → SEO → App Store优化 → 社媒策略
- 时长：6周

### 体系 4: AI 开发工程师
- 目标：能独立开发和部署 AI Agent 应用
- 课程：Agent 开发基础 → MCP 工具 → 前端开发 → DevOps
- 时长：8周

### 体系 5: 全栈开发工程师
- 目标：能独立完成 Web 应用的前后端开发
- 课程：前端开发 → 后端架构 → 数据库 → DevOps → 安全
- 时长：10周

### 体系 6: 项目管理专员
- 目标：能独立管理软件项目从立项到交付
- 课程：项目管理 → 产品管理 → 质量测试 → 运营支持
- 时长：6周

### 体系 7: UI/UX 设计师
- 目标：能独立完成产品的界面设计和用户体验优化
- 课程：UI设计 → UX研究 → UX架构 → 品牌守护 → 视觉叙事
- 时长：8周

### 体系 8: 销售精英
- 目标：能独立完成从获客到成交的全流程销售
- 课程：外呼策略 → 发现式销售 → 交易策略 → 提案撰写 → 管道分析
- 时长：6周
