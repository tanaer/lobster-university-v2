## 第一章：课程概述

### 学习目标
- 理解 Spec-First 任务定义
- 掌握两阶段审查流程
- 学会依赖图管理

### 为什么需要协作协议？
多 Agent 协作容易混乱，协议确保：
- 任务定义清晰
- 质量可控
- 依赖关系明确

---

## 第二章：核心概念

### 2.1 Spec-First 方法
先写规格说明，再执行任务。

### 2.2 两阶段审查
1. **Spec 审查** - 检查任务定义
2. **质量审查** - 检查执行结果

### 2.3 Beads 依赖图
使用有向无环图管理任务依赖。

---

## 第三章：基础用法

### 3.1 定义任务规格
```yaml
task:
  name: "实现用户认证"
  inputs: ["需求文档"]
  outputs: ["认证模块"]
  dependencies: []
```

### 3.2 分配任务
将任务分配给合适的 Agent。

### 3.3 执行审查
按两阶段流程审查结果。

## 第六章：最佳实践

### ✅ 推荐做法
- 任务粒度适中
- 明确定义接口
- 定期同步状态

### ❌ 避免做法
- 任务过于庞大
- 循环依赖
- 缺少审查

---

## 第七章：常见问题

### Q1: Agent 间通信延迟？
使用消息队列解耦。

### Q2: 任务失败影响其他任务？
设计隔离和重试机制。
