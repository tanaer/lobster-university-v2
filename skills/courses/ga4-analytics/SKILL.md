### 为什么学习 GA4？

Google Analytics 4 是 Google 推出的新一代分析工具，2023 年 7 月起完全替代 Universal Analytics。

| 对比维度 | Universal Analytics | GA4 |
|----------|---------------------|-----|
| 数据模型 | 基于会话和页面浏览 | 基于事件 |
| 跨平台 | 仅网站 | 网站+APP |
| 机器学习 | 有限 | 内置预测功能 |
| 隐私合规 | 依赖 Cookie | 更隐私友好 |
| 报告灵活性 | 预定义报告 | 探索报告自由组合 |

**市场数据**：90% 以上的网站使用 Google Analytics，GA4 技能是数据运营岗位的标配。

## 第二章：核心概念

### 2.1 GA4 事件模型

GA4 的核心是**事件（Event）**，一切用户行为都是事件：

```
用户行为 → 事件 → 参数 → GA4 后台
```

**事件类型**：

| 类型 | 说明 | 示例 |
|------|------|------|
| 自动收集 | 无需配置，自动追踪 | page_view, scroll, click |
| 增强测量 | 开启后自动收集 | 出站点击、站内搜索、视频互动 |
| 推荐事件 | Google 预定义，建议实现 | sign_up, purchase, generate_lead |
| 自定义事件 | 你自己定义 | 按钮点击、表单提交、特定交互 |

### 2.2 关键指标

| 指标 | 含义 | 使用场景 |
|------|------|----------|
| 活跃用户数 | 至少触发一个事件的独立用户 | 衡量流量规模 |
| 会话数 | 用户的一次访问（30分钟无活动结束）| 衡量访问频次 |
| 平均参与时间 | 用户与网站互动的平均时长 | 衡量内容质量 |
| 跳出率 | 无互动就离开的会话比例 | 衡量着陆页质量 |
| 转化率 | 完成目标事件的用户比例 | 衡量业务效果 |

### 2.3 GA4 账户结构

```
Google Analytics 账户
    └── 媒体资源（Property）— 对应一个网站/APP
        └── 数据流（Data Stream）— 网站或 APP
            └── 事件（Event）
```

**最佳实践**：
- 一个公司一个账户
- 一个产品一个媒体资源
- 网站和 APP 分别建数据流

## 第四章：基础用法

### 4.1 配置增强测量

进入 GA4 后台 → 数据流 → 网站 → 增强测量 → 开启：

- [x] 网页浏览（自动）
- [x] 滚动（滚动深度 90%）
- [x] 出站点击
- [x] 站内搜索
- [x] 视频互动
- [x] 文件下载

### 4.2 设置转化事件（关键！）

转化事件 = 你的业务目标

**常见转化事件**：

| 业务类型 | 转化事件 | 事件参数 |
|----------|----------|----------|
| 电商 | purchase | value, currency, items |
| SaaS | sign_up | method |
| 内容站 | generate_lead | 表单完成 |
| 工具站 | file_download | file_name |

**配置步骤**：
1. 确保事件已在发送（通过 DebugView 验证）
2. 进入 GA4 → 事件 → 标记为转化
3. 或者：管理 → 转化 → 新建转化事件

### 4.3 查看基础报告

**首页报告**：
- 用户获取概览
- 用户互动概览
- 事件计数
- 转化次数

**生命周期报告**：
- 获客：流量来源分析
- 互动：用户行为分析
- 变现：电商/收入分析
- 留存：用户回访分析

**用户报告**：
- 受众特征：年龄、性别、兴趣
- 地理位置：国家、城市
- 设备：操作系统、浏览器

## 第五章：进阶技巧

### 5.1 自定义事件追踪

**场景**：追踪"加入购物车"按钮点击

```javascript
// 代码实现
gtag('event', 'add_to_cart', {
  currency: 'CNY',
  value: 299.00,
  items: [{
    item_id: 'SKU_12345',
    item_name: '产品名称',
    item_category: '分类',
    quantity: 1,
    price: 299.00
  }]
});
```

**通过 GTM 实现（无代码）**：
1. 创建触发器：点击 - 某些点击 - Click Classes 包含 "add-to-cart"
2. 创建标签：GA4 事件 - 事件名称 "add_to_cart"
3. 添加参数：从点击元素提取商品信息

### 5.2 受众群体（细分用户）

**创建受众**：

| 受众类型 | 条件 | 用途 |
|----------|------|------|
| 高价值用户 | 购买次数 > 1 | 再营销 |
| 购物车放弃者 | 加购但未购买 | 挽回 |
| 新访客 | 首次访问 | 欢迎优惠 |
| 回访用户 | 7天内回访 | 忠诚度分析 |

**预测受众**（GA4 自动创建）：
- 购买可能性高的用户
- 流失风险高的用户
- 预测收入

### 5.3 漏斗分析

**创建漏斗**：
1. 探索 → 漏斗探索
2. 添加步骤：
   - 步骤 1：session_start（会话开始）
   - 步骤 2：view_item（查看商品）
   - 步骤 3：add_to_cart（加入购物车）
   - 步骤 4：begin_checkout（开始结账）
   - 步骤 5：purchase（购买）
3. 查看每一步的流失率

**漏斗洞察**：
- 哪个步骤流失最多？
- 不同流量来源的漏斗表现？
- 移动 vs 桌面的转化差异？

### 5.4 探索报告（自由分析）

**自由形式**：
- 行：流量来源
- 列：设备类别
- 值：转化次数

**路径探索**：
- 用户从首页去了哪里？
- 购买前的典型路径是什么？

**同类群组分析**：
- 按首次访问日期分组
- 看不同批次的留存率变化

## 第六章：最佳实践

### ✅ 推荐做法

1. **尽早设置转化事件**：安装 GA4 后立即标记关键业务事件
2. **使用一致的命名规范**：事件名用小写+下划线（add_to_cart）
3. **定期清理测试数据**：使用数据过滤器排除内部流量
4. **设置自定义维度**：如用户等级、会员类型
5. **导出到 BigQuery**：原始数据留存，高级分析
6. **与 Search Console 关联**：看搜索表现数据

### ❌ 避免做法

1. **追踪 PII（个人身份信息）**：邮箱、电话等，违反政策
2. **事件名混乱**：AddToCart、addToCart、add_to_cart 混用
3. **忽略测试**：上线前用 DebugView 验证事件
4. **只看汇总数据**：要细分受众、设备、来源分析
5. **不设置转化**：只看不行动，无法衡量效果

## 第七章：常见问题

### Q1: GA4 和 Universal Analytics 数据为什么不一致？
正常。两者数据模型不同：
- UA 基于会话，GA4 基于事件
- GA4 更隐私友好，会过滤更多机器人流量
- 趋势一致即可，绝对数值不必强求一致

### Q2: 如何排除内部流量？
管理 → 数据流 → 更多标记设置 → 定义内部流量 → 添加 IP 规则

### Q3: GA4 数据保留多久？
默认 2 个月，可设置延长至 14 个月。建议导出到 BigQuery 永久保存。

### Q4: 为什么我的事件没有显示？
检查清单：
- [ ] 代码是否正确安装
- [ ] 是否在 DebugView 中看到事件
- [ ] 事件是否已标记为转化（如果是转化）
- [ ] 等待 24-48 小时（实时报告有延迟）

### Q5: 如何追踪跨域访问？
配置 → 数据流 → 更多标记设置 → 配置您的网域 → 添加相关域名

### Q6: GA4 免费版有什么限制？
- 每月 1000 万事件（大部分网站够用）
- 50 个自定义维度
- 50 个自定义指标
- 400 个受众群体
