## 第二章：核心概念

### 2.1 AI 生成文本的特征

**语言特征**：

| 特征 | AI 表现 | 人类表现 |
|------|---------|----------|
| 词汇多样性 | 重复用词，多样性较低 | 词汇丰富，有个人风格 |
| 句子长度 | 倾向于平均长度 | 长短变化大 |
| 情感表达 | 中性、客观 | 有情感波动 |
| 逻辑结构 | 过于规整 | 有跳跃和联想 |
| 细节描述 | 泛泛而谈 | 具体生动 |

**常见 AI 写作模式**：

1. **过度使用连接词**："此外"、"值得注意的是"、"总的来说"
2. **三段式结构**：开头-中间-结尾过于标准
3. **列表泛滥**：喜欢用 1. 2. 3. 列举
4. **缺乏具体案例**：理论多，实例少
5. **结尾总结公式化**："综上所述"、"总而言之"

### 2.2 AI 检测原理

**基于统计的方法**：
- 困惑度（Perplexity）：AI 生成文本困惑度通常较低
- 突发性（Burstiness）：人类写作有高低起伏，AI 较平均

**基于机器学习的方法**：
- 在大量人类/AI 文本上训练分类器
- 提取文本特征进行判断
- 输出概率分数

**局限性**：
- 无法 100% 准确
- 随着 AI 进化，检测难度增加
- 人类写作可能被误判为 AI
- AI 写作经过人工修改后难检测

## 第四章：基础用法

### 4.1 人工识别方法

**第一步：整体感知**
- 阅读时是否有"机器感"？
- 是否过于流畅但缺乏灵魂？

**第二步：检查特征词**

AI 高频词（中文）：
- 值得注意的是
- 此外/另外
- 总的来说/综上所述
- 从某种程度上说
- 不可否认的是

AI 高频词（英文）：
- Delve into
- In conclusion
- It's important to note
- As a language model
- Furthermore

**第三步：分析结构**

```
AI 典型结构：
引言（定义+背景）
  ↓
主体（分点论述，每点一个方面）
  ↓
结论（总结+展望）

人类典型结构：
故事开头/观点抛出
  ↓
跳跃式论述（联想、举例、反驳）
  ↓
个人感悟/开放式结尾
```

**第四步：验证细节**
- 检查具体数据、日期、人名是否准确
- 看是否有个人经历或独特观点
- 搜索关键句子看是否网上有雷同

### 4.2 使用检测工具

**GPTZero 使用步骤**：
1. 访问 https://gptzero.me
2. 粘贴或上传文本
3. 点击"Check Origin"
4. 查看结果：
   - Human Written（人类撰写）
   - Mixed（混合）
   - AI Generated（AI 生成）

**解读结果**：

| 分数 | 含义 | 建议 |
|------|------|------|
| 0-20% | 很可能是人类 | 可信度高 |
| 20-60% | 可能混合 | 需要进一步验证 |
| 60-100% | 很可能是 AI | 需要人工复核 |

### 4.3 批量检测工作流

```python
# 批量检测示例
import os
import time

def batch_detect(folder_path):
    results = []
    
    for filename in os.listdir(folder_path):
        if filename.endswith('.txt'):

            # 调用检测 API
            score = detect_ai_content(content)
            results.append({
```

## 第五章：进阶技巧

### 5.1 提高检测准确率

**组合检测法**：
1. 先用工具快速筛选（高置信度结果直接采用）
2. 对中等置信度用第二种工具交叉验证
3. 对争议内容人工深度分析

**分段检测**：
- 长文章分段检测
- 不同段落可能有不同来源
- 关注段落间的风格突变

**历史对比**：
- 对比作者历史作品
- 突然的风格变化值得怀疑
- 建立作者写作指纹档案

### 5.2 识别 AI 改写

**常见改写手段**：
- 同义词替换
- 句子重组
- 添加个人化开头结尾
- 插入具体例子

**检测方法**：
1. 去掉开头结尾再检测
2. 提取核心段落检测
3. 检查逻辑连贯性（改写后可能不连贯）

### 5.3 处理误判

**人类被误判为 AI 的原因**：
- 写作风格规范、客观
- 使用了模板或结构化写作
- 非母语者写作（句式简单）

**AI 被误判为人类的原因**：
- 经过大量人工修改
- 使用了高级提示词技巧
- 混合了人类写作片段

**应对策略**：
- 不依赖单一工具
- 结合上下文判断
- 必要时与作者沟通

### 5.4 构建检测系统

**简单版本（Python）**：

```python
class AIDetector:
    def __init__(self):
        self.ai_patterns = [
            r'值得注意的是',
            r'此外|另外',
            r'总的来说|综上所述',
            r'从某种程度上说',
        ]
    
    def detect(self, text):
        score = 0
        
        # 模式匹配
        for pattern in self.ai_patterns:
            if re.search(pattern, text):
                score += 10
        
        # 句子长度分析
        sentences = text.split('。')
        avg_len = sum(len(s) for s in sentences) / len(sentences)
        if 15 < avg_len < 25:  # AI 倾向于中等长度
            score += 15
        
        # 词汇多样性
        words = text.split()
        unique_ratio = len(set(words)) / len(words)
        if unique_ratio < 0.5:  # AI 词汇多样性较低
            score += 20
        
        return min(score, 100)
```

## 第六章：最佳实践

### ✅ 推荐做法

1. **多工具交叉验证**：不要依赖单一工具的结果
2. **保留检测记录**：截图或导出报告存档
3. **结合上下文**：考虑提交时间、作者背景等因素
4. **定期更新知识**：AI 在进化，检测方法也要更新
5. **透明沟通**：向被检测者说明检测方法和局限性

### ❌ 避免做法

1. **完全依赖工具分数**：工具只是辅助，不是判决
2. **公开指责**：先私下沟通，给解释机会
3. **忽视误判可能**：保持谦逊，承认不确定性
4. **一刀切政策**：根据场景制定合理标准
5. **忽视隐私**：注意文本内容的敏感性

## 第七章：常见问题

### Q1: AI 检测工具的准确率有多高？
没有工具能做到 100% 准确。主流工具在理想条件下准确率约 70-90%，但受文本类型、长度、AI 模型版本影响。短文本（<100 字）检测准确率显著下降。

### Q2: 检测结果是 80% AI，能确定是 AI 写的吗？
不能 100% 确定。80% 表示"很可能是 AI"，建议：
1. 用第二个工具验证
2. 人工分析关键特征
3. 必要时与作者沟通

### Q3: 学生用 AI 写论文然后大量修改，能检测出来吗？
很难。如果修改程度高（改写 50% 以上），检测工具基本无法识别。这种情况需要：
- 要求写作过程稿
- 口头答辩
- 课堂小测验证知识掌握

### Q4: 不同检测工具结果不一致怎么办？
正常。建议：
- 取多个工具的平均值
- 关注置信度高的结果
- 对争议内容人工判断

### Q5: AI 检测会侵犯隐私吗？
取决于使用场景：
- 学术检测：通常事先告知，合理
- 私下监控：可能侵犯隐私
- 公开内容检测：一般无问题

### Q6: 未来 AI 检测还有意义吗？
有意义，但角色会变化：
- 从"抓作弊"转向"质量评估"
- 从"判断来源"转向"内容审核"
- 人机协作检测将成为主流
